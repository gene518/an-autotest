# 静默导医功能测试计划

## Application Overview

基于北大医生IM聊天页面静默导医功能的综合测试计划，验证理赔用户在特定场景下（问诊结束为重症推荐、咨询医院时）静默导医功能的正确触发逻辑。

## Test Scenarios

### 1. 静默导医场景验证

**Seed:** `seed.spec.ts`

#### 1.1. 问诊结束为重症时静默导医（癌症推荐）

**File:** `silent_referral/critical_illness_referral.spec.ts`

**测试配置:**
- 测试超时时间：3分钟（180000ms）
- **原因:** 问诊流程需要多轮AI交互

**适用对象:** 理赔用户

**Steps:**
  1. 访问登录并跳转到IM页面（使用理赔用户账号）
     - expect: 登录成功并自动跳转到IM页面
     - expect: 页面成功加载并显示AI医生标题
  2. 等待页面完全加载
     - expect: 顶部导航栏显示'对话'和'我的'标签
     - expect: 聊天输入框正常显示
  3. 点击右上角开启新对话图标
     - expect: 显示欢迎话术
  4. 输入与癌症/重大疾病相关的问诊内容并发送（如：我最近被确诊为肺癌，该怎么办？）
     - expect: 输入内容发送到对话中
     - expect: AI进入问诊流程
  5. 与AI进行多轮问诊交互，选择选项或回复不清楚
     - expect: 每轮AI回复正常
  6. 等待问诊结束，AI判断为重症
     - expect: 问诊出现健康小结
     - expect: AI识别为重症（如癌症）
  7. 验证静默导医触发
     - expect: 系统静默推荐相关科室或医院（对用户无感知的导医操作）
     - expect: 页面出现癌症相关推荐卡片或信息

#### 1.2. 用户咨询医院时静默导医

**File:** `silent_referral/hospital_inquiry_referral.spec.ts`

**适用对象:** 理赔用户

**Steps:**
  1. 访问登录并跳转到IM页面（使用理赔用户账号）
     - expect: 登录成功并自动跳转到IM页面
  2. 等待页面完全加载
     - expect: 聊天输入框正常显示
  3. 点击右上角开启新对话图标
     - expect: 显示欢迎话术
  4. 输入咨询医院相关的内容并发送（如：哪家医院治疗肺癌比较好？）
     - expect: 输入内容发送到对话中
     - expect: AI回复医院相关建议
  5. 验证静默导医触发
     - expect: 系统在用户咨询医院的同时静默推荐医院
     - expect: 相关推荐内容展示符合预期，用户体验流畅
